# Генератор QR-кодов

Простой статический сайт на базе `qrcode.js`, адаптированный под:
- телефон;
- планшет;
- компьютер.

Возможности:
- ввод ссылки и генерация QR по кнопке;
- автоматическое добавление `https://`, если протокол не указан;
- опциональная очистка UTM/трекеров и `#` якоря;
- скачивание в `PNG`, `SVG`, `JPG`, `PDF`.

## Почему лучше не убирать `https://`

Для корректного открытия ссылки после сканирования рекомендуется кодировать **полный URL**.
Если убрать `http://` или `https://`, часть сканеров может показать строку как текст,
а не открыть сайт автоматически.

## Локальный запуск

Откройте `index.html` в браузере.

## Публикация на GitHub Pages

1. Загрузите изменения в ветку `main`.
2. В GitHub: **Settings → Pages**.
3. В поле **Build and deployment** выберите:
   - **Source**: `Deploy from a branch`
   - **Branch**: `main` и папка `/ (root)`
4. Сохраните настройки.
5. Через 1–2 минуты сайт будет доступен по адресу вида:
   `https://<username>.github.io/<repo>/`

## Важное про «вечность» QR

QR-код хранит текст ссылки. Это значит:
- сам QR можно хранить бесконечно;
- работать он будет, пока доступна ссылка внутри него.

Чтобы QR-ссылки жили максимально долго:
1. Используйте домен, который контролируете сами.
2. Не меняйте структуру URL после публикации.
3. По возможности используйте постоянный редирект-домен
   (например `go.yourdomain/...`) и меняйте целевой адрес на сервере,
   не изменяя сам QR.
4. Держите резервную копию сайта и DNS-настроек.

## Лицензии и авторство

`qrcode.js` — сторонняя библиотека с собственной лицензией. Удалять из неё
исходные упоминания авторов и лицензии не рекомендуется.
Безопасный путь: оставлять библиотеку как внешний vendor-компонент,
а свой интерфейс и бизнес-логику развивать в собственных файлах.
